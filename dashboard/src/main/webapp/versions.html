4.1 Released June 26, 2014

Documentation

Release Notes

OpenGeo Suite upgrades

  • Improved enterprise packages. Whether developing applications on OSX or deploying a Windows production system, our packages are designed to help you use our software more effectively. Now enterprise customers benefit from other typical IT deployment options like supported deployments for Amazon Web Services and VMware.
  • Automated clustering. We're continually improving our clustering features for those needing high availability or better scaling under load. These improvements make spinning up clusters even easier for systems administrators by providing new automation options.
  • New data sources. OpenGeo Suite can now publish from MongoDB and GeoPackage.
  • New Boundless SDK templates that use a production-ready version of OpenLayers 3. These free templates make it easier and faster to deploy enterprise applications built using OpenGeo Suite.
  • QGIS 2.2 one-click installers. Dramatically increase the ease and affordability of desktop analysis and editing tools with our bundled QGIS that includes the OpenGeo Explorer plugin to publish maps and data directly to OpenGeo Suite.
  • Mapmeter now included with all Enterprise tiers. Our monitoring service helps developers, system administrators, and managers better understand and use their geospatial deployments so we're including the service for all OpenGeo Suite enterprise customers.

This maintenance release also includes Component upgrades:

PostgreSQL 9.3.4

  • Materialized views, great for caching summary information
  • Enhanced JSON and key-value support, for better non-relational modelling
  • Writeable foreign tables, for federated databases
  • Replication enhancements, for database clusters

PostGIS 2.1.3

  • Performance improvements in geography, 2-d indexing, and query planning
  • Performance improvements in raster computations and geocoding
  • Many more and faster raster analysis functions
  • Better support for curved geometry types
  • Delaunay triangulation and enhanced 3D object support

PointCloud LIDAR 0.1.0

  • Store and analyze multi-dimensional LIDAR data
  • 4:1 lossless compression
  • Filter on attributes, spatial clip and union data for output
  • Integration with PostGIS for advanced analysis
  • Read and write from multiple LIDAR formats with PDAL loader

GeoServer 2.5.1

  • New standards support for Web Coverage Service (WCS) 2.0 and WCS 2.0 Earth Observation
  • Batch importer for easier bulk data loading
  • High performance PNG encoder based on PNGJ library
  • Improved PostGIS rendering performance using ST_Simplify()
  • Improved GetFeatureInfo implementation take the shape of the clicked target into consideration, for easier feature identification
  • Improved support for LDAP groups
  • Improved performance in large JDBC clustering implementations

GeoWebCache 1.5.2

  • Transparent GeoServer integration for all WMS request types
  • Support for longer layer names and GeoServer JDBC catalogue
  • Enhance support for REST configuration

OpenLayers 3.0PRE

  • Preview of upcoming OpenLayers 3.0 release
  • Tiled Layers: Pull tiles from OSM, Bing, MapBox, Stamen, MapQuest, and any other XYZ source you can find. OGC mapping services and untiled layers also supported.
  • Vector Layers: Render vector data from GeoJSON, TopoJSON, KML, GML, and a growing number of other formats.
  • Fast & Mobile Ready: Mobile support out of the box. Build lightweight custom profiles with just the components you need.
  • Cutting Edge & Easy to Customize: Map rendering leverages WebGL, Canvas 2D, and all the latest greatness from HTML5. Style your map controls with straight-forward CSS.

Components included:

  • PostgreSQL 9.3.4
  • PostGIS 2.1.3
  • GeoServer 2.5.1
  • GeoTools 11 ("These go to 11")
  • GeoWebCache 1.5
  • OpenLayers 3
  • QGIS 2.2

4.0.2 Released January 23, 2014

Documentation

Release Notes

OpenGeo Suite upgrades

  • Completely new packages. Whether developing applications on OSX or deploying a Windows production system, our packages are designed to help you use our software more effectively.
  • Scalability and clustering. We now offer a clustering extension for GeoServer for those needing high availability or better scaling under load.
  • Improved support for LIDAR point clouds. PostGIS can now natively store and filter LIDAR data.

  • OpenLayers 3 is a complete rewrite that simplifies the developer experience and takes advantage of modern JavaScript environments to offer a high-performance and fully-featured web mapping library. See how it compares to Google Maps API or GeoExt.
  •  OpenGeo Suite Exporer Plugin for QGIS allows users to seamlessly manage and publish from the desktop to OpenGeo Suite.

Documentation

Release Notes

This release includes the following improvements and Component upgrades:

PostgreSQL 9.3

  • Materialized views, great for caching summary information
  • Enhanced JSON and key-value support, for better non-relational modelling
  • Writeable foreign tables, for federated databases
  • Replication enhancements, for database clusters

PostGIS 2.1

  • Performance improvements in geography, 2-d indexing, and query planning
  • Performance improvements in raster computations and geocoding
  • Many more and faster raster analysis functions
  • Better support for curved geometry types
  • Delaunay triangulation and enhanced 3D object support

PointCloud LIDAR support

  • Store and analyze multi-dimensional LIDAR data
  • 4:1 lossless compression
  • Filter on attributes, spatial clip and union data for output
  • Integration with PostGIS for advanced analysis
  • Read and write from multiple LIDAR formats with PDAL loader

GeoServer 2.4

  • Catalog Services as an extension for the Web (CSW) 2.0.2 implementation with support for the ISO Metadata Application Profile next to Dublin Core metadata
  • Clustering extensions that make management of a GeoServer cluster much easier by storing the GeoServer catalog in an external database as well as providing a messaging layer that automates node synchronization on configuration change events
  • CSS-like styling tools included as an extension
  • Editor for GeoWebCache parameter filters
  • Improved rendering of complex point symbols in KML

GeoWebCache 1.5

  • Metastore removed
  • Ability to create meaningful names for capabilities files
  • Pluggable URL Mangling
  • Disk quota over JDBC
  • List layers through REST API
  • Configurable GetInfo MIME formats
  • New ParameterFilter types can be added easily
  • XStream configuration can be varied between REST and Config file contexts
  • REST endpoint for mass truncating layers
  • Documentation improvements

Support for OpenLayers 3.0

  • While our packages ship with OpenLayers 2.12, our customers also receive early access and support for OpenLayers 3, which includes many new features:
  • Tiled Layers: Pull tiles from OSM, Bing, MapBox, Stamen, MapQuest, and any other XYZ source you can find. OGC mapping services and untiled layers also supported.

  • Cutting Edge & Easy to Customize: Map rendering leverages WebGL, Canvas 2D, and all the latest greatness from HTML5. Style your map controls with straight-forward CSS.
